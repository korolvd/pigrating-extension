# PigRating — соцрейтинг свиноколеса

Chrome-расширение для [pointauc.com](https://pointauc.com/): перед прокрутом применяет баллы соцрейтинга из Google-таблицы к лотам через официальный публичный API (`/api/oshino`). Дополнительно умеет **принимать покупку рейтинга за балы канала Twitch** — начисления пишутся в ту же таблицу (см. раздел [ниже](#покупка-рейтинга-за-балы-канала-twitch)).

## Как это работает

1. Читает Google-таблицу (CSV-экспорт). **Ссылку бери прямо из адресной строки** браузера на нужной вкладке. Для своих таблиц работает, пока ты залогинен в Google в этом Chrome (для чужой/без логина — дай таблице доступ «по ссылке: просмотр»). Где брать ник и баллы — задаётся в настройках: **номер первой строки данных**, **столбец ника** и **столбец баллов**.
2. Берёт лоты через `GET /api/oshino/lots` (каждый лот содержит список вкладчиков `investors`).
3. Сопоставляет ник из таблицы с вкладчиком лота (без учёта регистра) и решает, что делать:

| Ситуация | Действие |
|---|---|
| Ник — единственный реальный вкладчик одного лота | **автоматически** «+ к лоту» |
| Все остальные случаи: нет лота / групповой лот / ник в нескольких лотах | **выбор лота в окне** — в предпросмотре у строки выпадающий список: существующий лот, «новый лот» или «пропустить» (по умолчанию) |

Балы всегда применяются **ставкой** (`POST /bids`) от инвестора-метки `[СОЦРЕЙТИНГ] ник:сумма` (сумма зашита в имя — нужна для отката):
- «+ к лоту» / выбранный лот → ставка с `message: "#fastId"`, `insertStrategy: "match"` (точно в этот лот);
- «новый лот» → ставка с `message: "[СОЦРЕЙТИНГ] ник"`, `insertStrategy: "force"`.

Поэтому на карточке видно подпись «от [СОЦРЕЙТИНГ] ник:сумма» — какие балы пришли от рейтинга. Эти метки **не учитываются** при сопоставлении (чтобы лот не стал «групповым» на следующем прогоне).

**Защита от задвоения:** если у лота уже есть метка этого ника, строка помечается «⚠ уже применено» и по умолчанию пропускается (можешь всё равно выбрать лот вручную или откатить инлайн-кнопкой `↩`).

**Откат** работает по меткам прямо с доски (стейтлес): сумма метки `:N` вычитается из лота, метка снимается. Доступен в любой момент, не зависит от памяти расширения.

Авто-создания лотов нет: новый лот появляется только если явно выбрать «новый лот». Отрицательные баллы (при галке «разрешать минус») вычитаются/применяются так же; нулевые пропускаются.

## Установка

1. `chrome://extensions` → включить **Режим разработчика**.
2. **Загрузить распакованное расширение** → выбрать эту папку.
3. Открыть popup, в **⚙️ Настройки** вписать:
   - **Personal Token** — `pointauc.com` → Настройки → раздел *Personal Token*;
   - **Ссылку на таблицу** — скопируй прямо из адресной строки браузера на нужной вкладке (кнопка «Поделиться» не нужна). Для своих таблиц достаточно быть залогиненным в Google в этом Chrome.
4. Сохранить.

## Использование

- **🔍 Предпросмотр** — показывает план без изменений (что куда зальётся, что пропустится). Здесь же проверяешь, что ники из таблицы реально матчатся с вкладчиками лотов. Строки отсортированы: сперва авто «+ к лоту», затем «много лотов», «групповой лот», «без лота», в конце «уже применено» и «пропуск».
- **🎰 Применить** — выполняет план. Требует подтверждения (повторный клик).
- **↩ Откатить** — сканирует доску и показывает все метки рейтинга с галочками: `Снять выбранные` или `Снять все`. Сумма метки вычитается из лота, метка удаляется. Конкретного ника можно откатить инлайн-кнопкой `↩` на строке «⚠ уже применено» (и в таблице-результате после залива).

⚠️ Балы **прибавляются** (аддитивно). Повторный запуск **удвоит** балы (но «уже применено» это ловит). Откат — единственный безопасный способ убрать рейтинг.

## Настройки

- **1-я строка данных** — с какой строки начинать читать (по умолчанию `2`, т.к. строка 1 — заголовок).
- **Столбец ника / столбец баллов** — буквой (`A`, `B`, …) или номером (`1`, `2`, …). По умолчанию `A` и `B`.
  - ⚠️ **Ники в таблице должны совпадать с никами на Twitch.** Сопоставление идёт по `investorId`, а для channel points это twitch-логин — иначе ник не сматчится с лотом. Регистр и ведущий `@` не важны (при записи покупок ник нормализуется, новый ник добавляется сразу после последнего ника столбца).
- **Префикс подписи инвестора** — балы применяются от инвестора `[СОЦРЕЙТИНГ] ник` (приставка по умолчанию `[СОЦРЕЙТИНГ] `), поэтому на лоте видно «от [СОЦРЕЙТИНГ] ник». Эта же приставка идёт в имя при выборе «новый лот». Инвесторы с такой приставкой не учитываются при сопоставлении и используются для детекта «уже применено».
- **Разрешать минус** — обрабатывать отрицательные баллы (по умолчанию вкл.): вычитать у одиночного лота / разрешать в списке. Выключено — минусы полностью игнорируются.
- **Пропускать нули** — строки с `0` игнорируются (по умолчанию вкл.).
- **Как донат** — слать ставки как донат (`isDonation: true`): карточка в донат-стиле, и к сумме применяется конвертация аука «деньги→баллы». По умолчанию **выкл.** — баллы идут 1:1, конвертация игнорируется. ⚠️ При конвертации ≠ 1:1 откат вычтет «сырое» число из метки, а не реально начисленное (pointauc умножил) — для точного отката держи конвертацию 1:1 или не используй донат-режим.

## Покупка рейтинга за балы канала Twitch

Зрители тратят баллы канала Twitch и меняют соцрейтинг — начисления пишутся в **ту же Google-таблицу**, которую расширение потом применяет к лотам.

**Запись в таблицу.** Таблица доступна на чтение по ссылке, но писать в неё может только владелец. Поэтому запись идёт через **Google Apps Script Web App** (бесплатно, без Google Cloud): один раз ставишь скрипт в свою таблицу, он принимает `POST {secret, nick, points}` и инкрементит баллы ника. Кнопка **«📋 Скопировать скрипт»** собирает код с твоими настройками (лист, столбцы) и генерит секрет; пошаговая инструкция со скриншотами — кнопка **«❔ Как установить?»**. Индикатор у поля URL показывает 🟢 / 🔴.

**Подключение Twitch.** Кнопка **«Подключить Twitch»** — OAuth (implicit), приложение встроено в расширение (своё регистрировать не нужно), скоуп `channel:manage:redemptions`.

**Награды.** В таблице наград задаёшь строки **название · цена в балах · рейтинг (можно минус) · кому**. Расширение само создаёт/обновляет награды на канале (нужен статус **Affiliate/Partner**). Мастер-тумблер **«Награды за балы канала»** наверху: вкл — создаёт и включает награды, выкл — отключает (не удаляет).

**Кому начисляется:**
- **себе** — баллы зрителю, который купил (награда **без поля ввода**);
- **ник из ввода** — нику из поля награды (подарок другу или диверсия — минус-награда). ⚠️ Twitch делает это поле **обязательным** — «необязательного» ввода у наград в API нет. Поэтому для награды «для себя» ставь «себе», для подарка — «ник из ввода» (а не одну награду на оба случая).

Ник в поле можно вводить с `@` или без, регистр не важен (`@ReFiRnE` = `refirne`).

**Начисление и подтверждение.** По умолчанию работает **авто-режим** (тумблер **«auto» ⚡** в шапке очереди): покупки себе, на **точных участников таблицы** и на **совсем новые ники** (без похожих в таблице) начисляются сразу. В очередь **«На подтверждение»** уходят только ники, **похожие** на кого-то из таблицы (вероятная опечатка) — там стример решает: **✓ начислить** на введённый (новый участник), **✕ вернуть баллы** или клик по подсказке (см. ниже). Выключишь «auto» — на подтверждение пойдут все покупки. В логе видно `@покупатель → ник`; при ошибке записи баллы зрителю возвращаются автоматически, а если возврат не прошёл — это тоже видно в логе. Слушатель работает в фоне (EventSub WebSocket) с авто-переподключением.

**Подсказка похожего ника.** Если введённого ника нет в таблице (опечатка или другое написание — напр. `xsalreen` вместо `xalreenstream`), в очереди показывается ближайший ник из таблицы: **«возможно: xalreenstream»**. Клик начисляет на него (в логе — `исправлено: …`), а не на введённый. Сходство — коэффициент Сёренсена–Дайса по биграммам, как «похожий лот» в pointauc. Если таблица недоступна, подсказок нет и работает прежний гейт по наличию ника на Twitch.

## Что стоит проверить на боевом аукционе

Совпадает ли `investorId` в pointauc (для channel points это обычно twitch-логин) с ником в таблице — для этого и нужен **Предпросмотр**.

Если ник реально вкладывался в лот, но в предпросмотре оказался в группе **«выбрать лот»** с пометкой «нет лота» и без кандидатов (★) — значит, его ник в таблице не совпал с `investorId` в pointauc (другой регистр/формат). Тогда поправь ник в таблице или направь баллы в нужный лот вручную через выпадающий список.

## Тестирование (без зрителей)

Чтобы попробовать расширение без живого аукциона, в репозитории есть `test-lots.bat` (Windows) — он заливает на доску тестовые лоты с нужными инвесторами.

1. Открой `test-lots.bat` в редакторе.
2. Впиши свой токен в строку `set "TOKEN=..."`.
3. В блоке «СПИСОК ЛОТОВ» задай лоты строками `call :add "Название" "ник" баллы`:
   - один и тот же лот с разными никами → **групповой лот**;
   - один ник в разных лотах → **«ник в нескольких лотах»**;
   - ник, которого нет ни в одном лоте → в расширении **«выбрать лот»**.
   Имена лотов — **латиницей**; ники = ники из таблицы (twitch-логины).
4. Запусти двойным кликом — на каждый лот в окне появится `[201]`.
5. Открой pointauc, проверь доску. Дальше в расширении: **Предпросмотр** → сверь сопоставление → **Залить**.

Полезное:
- **Превью без отправки:** убери `REM` со строки `set "DRYRUN=1"` — покажет, что отправилось бы.
- Заливка аддитивна: перед новым прогоном **очисти доску** в pointauc (в API нет удаления; `force` объединяет лоты по имени, иначе суммы сложатся).

## Архитектура заливки

Заливка выполняется в **фоновом service worker** (`background.js`), а не в popup. Поэтому окно расширения можно закрыть / переключиться на вкладку аука — заливка не оборвётся и дойдёт до конца. Всё применяется **одним `POST /bids`** (ставки с инвестором-меткой): «+ к лоту» через `#fastId`+`match`, «новый лот» через `force`.

## Файлы

- `manifest.json` — MV3-манифест: права на `pointauc.com`, `docs.google.com`, `script.google.com`, Twitch (`api.twitch.tv` / `id.twitch.tv`), разрешения `identity`/`alarms`; фиксированный `key` (постоянный ID расширения для одного Twitch Redirect URL).
- `popup.html` / `popup.css` — интерфейс (статус-строка, действия, очередь покупок, секции настроек, SVG-иконки).
- `popup.js` — UI: предпросмотр/применение, откат, настройки, Twitch (подключение, награды, очередь подтверждений).
- `core.js` — общая логика: парсинг таблицы, API pointauc, план (батчем), запись баллов в таблицу (`addPoints` / `healthCheck`), генерация Apps Script (`buildAppsScript`), маппинг наград (`resolveRedemption`).
- `background.js` — фоновый worker: применение/откат независимо от окна **+ слушатель EventSub** (покупка рейтинга, очередь подтверждений).
- `twitch.js` — Twitch: OAuth (implicit), Helix (награды/редемпшены), подписка EventSub.
- `apps-script.gs` — образец скрипта для Google-таблицы (приём начислений). Обычно генерится кнопкой «Скопировать скрипт» с подставленными настройками.
- `help.html` + `help/` — инструкция по установке Apps Script со скриншотами (кнопка «❔ Как установить?»).
- `test/logic.test.mjs` — тесты (`npm test`).
- `icons/` — иконки расширения 16/48/128 (генерируются `npm run icons` из `tools/make-icons.mjs`).
- `test-lots.bat` — Windows-скрипт для заливки тестовых лотов (см. «Тестирование»).
