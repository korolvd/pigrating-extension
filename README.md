# PigPoints — свинобалы для свиноколеса

**PigPoints** — Chrome-расширение для [pointauc.com](https://pointauc.com/): превращает значки Twitch, балы канала и Google-таблицу в **ставки в лотах** через официальный публичный API (`/api/oshino`). Две механики:

- **Ставка за фильм (значки + PigPoints)** — флагман: зритель за 1 балл канала предлагает фильм, ставка = база + цены его значков Twitch + его PigPoints из таблицы. Уходит в очередь подтверждения pointauc. [Подробнее](#ставка-за-фильм-значки--pigpoints).
- **Покупка PigPoints за балы канала** — зрители тратят балы канала и меняют свои PigPoints; копятся в Google-таблице и питают ставку за фильм. [Подробнее](#покупка-pigpoints-за-балы-канала).

> **PigPoints** — «социальный рейтинг» зрителя из таблицы. Раньше расширение умело вручную заливать эти баллы в лоты (модуль PigRating) — ручной залив **выпиливается**, его заменяет автоматическая ставка за фильм. В коде он помечен `@deprecated`.

## Подключения

В **⚙️ Настройки → Подключения**:
- **pointauc Personal Token** — `pointauc.com` → Настройки → раздел *Personal Token*.
- **Twitch** — кнопка «Подключить Twitch» (OAuth implicit; приложение встроено в расширение, своё регистрировать не нужно).
- **Google-таблица (чтение)** — ссылку скопируй прямо из адресной строки браузера на нужной вкладке (кнопка «Поделиться» не нужна). Для своих таблиц достаточно быть залогиненным в Google в этом Chrome; для чужой — дай доступ «по ссылке: просмотр». Задаёшь: имя листа, 1-ю строку данных, столбец ника, столбец баллов.
- **Запись покупок (Apps Script)** — см. [ниже](#покупка-pigpoints-за-балы-канала).

⚠️ **Ники в таблице должны совпадать с twitch-логинами зрителей** — PigPoints ищутся по логину (регистр и ведущий `@` не важны).

## Ставка за фильм (значки + PigPoints)

Зритель тратит **1 балл канала** на награду **«Предложить фильм»**, вписывает название — в pointauc уходит ставка:

```
ставка = база + Σ(цены значков зрителя) + его PigPoints из таблицы
```

Уходит в **очередь подтверждения pointauc** (`insertStrategy: none`) — как донат: ты принимаешь её сам, pointauc предлагает похожий лот (свой матч лотов не нужен). Включается тумблером модуля **«Значки + PigPoints → ставка за фильм»** (создаёт награду на канале).

**Значки → вес.** В настройках «Значки и награды» задаёшь базу за активацию и **пул значков с ценами**: саб по тирам 1/2/3, VIP, модератор, фолловер, топ-1/2/3 даритель, топ-1/2/3 клипер, топ-1/2/3 по битам, sub-gifter по тирам, биты по тирам, founder.
- **саб (тир), VIP, мод, фолловер** — точечный запрос Helix в момент активации (свежо — ловит подписки «ради участия» по ходу аукциона);
- **топ-даритель (`sub-gift-leader`), топ-клипер (`clips-leader`), биты-лидер (`bits-leader`), sub-gifter/биты по тирам, founder** — из чат-значков сообщения награды (точечного API нет). ⚠️ В чате Twitch показывается лишь 3–4 значка, зритель сам выбирает какие — см. «анти-повтор».

**PigPoints → вес.** Галка «прибавлять PigPoints зрителя из таблицы» (по умолчанию вкл). Правило для знака **забетонировано** (стримеру выбор в конфликтах не даётся):
- **плюс** — прибавляется всегда (поддув в чужой фильм = хороший позыв);
- **минус** — по галке «не учитывать отрицательные PigPoints в общих лотах» (по умолчанию вкл): минус идёт только если зритель создаёт **новый** лот или он **единственный реальный вкладчик** найденного лота; в общий лот (поддув) минус не учитывается. Выключишь галку — минус учитывается везде.

«Свой/чужой» определяется по **текущей доске pointauc**: фаззи-матч названия фильма по именам лотов (метрика Сёренсена–Дайса, как «похожий лот» в pointauc) + список вкладчиков лота (метки `[PP]` не считаются). Нет в таблице / таблица недоступна → PigPoints не учитываются.

**Анти-повтор.** За раунд значки и PigPoints зрителя считаются **один раз**; повтор без новых значков → возврат балла. За несколько активаций (переключая показанный в чате значок) зритель добирает значки сверх лимита слотов — без двойного счёта. Учёт раунда сбрасывается **выключением/включением тумблера**. Из чего сложилась ставка (база · значки · PigPoints, 🟢 свой / 🟡 поддув) и что не учлось — видно в **журнале ставок**.

**Права Twitch.** Нужны скоупы `channel:manage:redemptions`, `user:read:chat`, `channel:read:subscriptions`, `channel:read:vips`, `moderation:read`, `moderator:read:followers` — при первом включении фичи нажми «Подключить Twitch» заново.

## Покупка PigPoints за балы канала

Зрители тратят балы канала Twitch и меняют свои PigPoints — начисления пишутся в **ту же Google-таблицу**, из которой берётся вес для ставки за фильм. Модуль **«PigPoints · таблица»** (свёрнут по умолчанию).

**Запись в таблицу.** Таблица доступна на чтение по ссылке, но писать может только владелец — поэтому запись идёт через **Google Apps Script Web App** (бесплатно, без Google Cloud): один раз ставишь скрипт в свою таблицу, он принимает `POST {secret, nick, points}` и инкрементит баллы ника. Кнопка **«📋 Скопировать скрипт»** собирает код с твоими настройками (лист, столбцы) и генерит секрет; пошаговая инструкция со скриншотами — **«❔ Как установить?»**. Индикатор у поля URL: 🟢 / 🔴 (и ловит «устаревший скрипт», если столбцы/лист в деплое разошлись с настройками).

**Награды.** В таблице наград задаёшь строки **название · цена в балах · PigPoints (можно минус) · кому**. Расширение само создаёт/обновляет награды на канале (нужен статус **Affiliate/Partner**). Тумблер **«Покупка PigPoints»** в модуле: вкл — создаёт и включает награды, выкл — отключает (не удаляет). Поменял награды → выключи и включи.

**Кому начисляется:**
- **себе** — зрителю, который купил (награда **без поля ввода**);
- **ник из ввода** — нику из поля награды (подарок другу или диверсия — минус-награда). ⚠️ Twitch делает поле ввода **обязательным**, поэтому для «себе» и для подарка делай **разные** награды.

Ник в поле можно вводить с `@` или без, регистр не важен (`@ReFiRnE` = `refirne`).

**Начисление и подтверждение.** По умолчанию **авто-режим** (тумблер **«auto» ⚡** в шапке очереди): покупки себе, на точных участников таблицы и совсем новые ники начисляются сразу. В очередь **«На подтверждение»** уходят только ники, **похожие** на кого-то из таблицы (вероятная опечатка) — там стример: **✓ начислить** / **✕ вернуть баллы** / клик по подсказке. В логе видно `@покупатель → ник`; ошибки возврата видны там же. Кнопка **«очистить журнал»** чистит лог начислений. Слушатель работает в фоне (EventSub WebSocket) с авто-переподключением.

**Подсказка похожего ника.** Введён ник не из таблицы (опечатка `xsalreen` вместо `xalreenstream`) → в очереди показывается ближайший ник: **«возможно: …»**. Клик начисляет на него (в логе — «исправлено»), а не на введённый. Сходство — коэффициент Сёренсена–Дайса по биграммам.

**Отдельный столбец покупок.** Галка «балы покупок — в тот же столбец»: выключишь — покупки пишутся в отдельный столбец и **в ставку за фильм не идут** (она читает основной столбец баллов).

## Тестирование (без зрителей)

`test-lots.bat` (Windows) заливает на доску pointauc тестовые лоты с нужными вкладчиками — удобно проверить детект «свой/чужой» для минуса PigPoints (единственный вкладчик / групповой лот).

1. Открой `test-lots.bat` в редакторе, впиши токен в строку `set "TOKEN=..."`.
2. В блоке «СПИСОК ЛОТОВ» задай лоты строками `call :add "Название" "ник" баллы` (имена лотов латиницей; ники = twitch-логины).
3. Запусти двойным кликом — проверь доску в pointauc.
- **Превью без отправки:** убери `REM` со строки `set "DRYRUN=1"`.

## Файлы

- `manifest.json` — MV3-манифест: права на `pointauc.com`, `docs.google.com`, `script.google.com`, Twitch (`api.twitch.tv` / `id.twitch.tv`), разрешения `identity`/`alarms`; фиксированный `key` (постоянный ID расширения → стабильный Twitch Redirect URL).
- `popup.html` / `popup.css` / `popup.js` — интерфейс: блок «Источники ставок» (модули), очередь покупок, журналы, настройки, SVG-иконки.
- `core.js` — общая логика: парсинг таблицы, API pointauc, значки→вес (`applicableMovieBadges`), PigPoints в ставке (`findMovieLot` / `moviePointsDecision`), запись в таблицу (`addPoints` / `healthCheck`), генерация Apps Script (`buildAppsScript`), маппинг наград (`resolveRedemption`). ⚠️ Функции ручного залива (`buildPlan` / `executePlan` / `buildRollbackPlan` / …) помечены **`@deprecated`** (на удаление).
- `background.js` — фоновый worker: слушатель **EventSub** (покупка PigPoints + ставка за фильм), очередь подтверждений, движок ставки за фильм.
- `twitch.js` — Twitch: OAuth (implicit), Helix (награды/редемпшены/статусы), подписка EventSub.
- `apps-script.gs` — образец скрипта приёма начислений (обычно генерится кнопкой «Скопировать скрипт»).
- `help.html` + `help/` — инструкция по установке Apps Script со скриншотами.
- `test/logic.test.mjs` — тесты (`node test/logic.test.mjs`); `icons/` — иконки 16/48/128 (`npm run icons`); `test-lots.bat` — тестовые лоты.
